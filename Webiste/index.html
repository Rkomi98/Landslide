Step 0: Dataset



There were three main types of data provided for this project:

  • The geopackage containing the vector layer of each group , from which we had to select ours (Group 2) and use it as a mask to clip the other data provided.
  • A list of environmental factor layers containing:
    1. Landslide Inventory
    2. DTM - Digital Terrain Model (from which the NLZ, Aspect, Slope and Plan and Profile Curvature will be derived)
    3. DUSAF – Land Use and Land Cover
    4. Distance to Features (i.e. buffers) of:
      1. Roads
      2. Geological Faults
      3. Rivers
    5. NDVI
  • Population Map. A spatial raster dataset with the estimated total number of people per grid-cell.

Step 1: Data Preprocessing for Susceptibility Mapping



The first step was perhaps the heftier and more operation dense out of all the different parts of the project. The various passages had to be executed almost fastidiously and here we found most of the problems encountered in the whole process. All of the following operations were done on the FOS software QGIS.

The first thing that had to be done after having obtained the data was to clip the different layers using as a mask our group’s sub-area. After this operation was completed for all the layers, we also had to resample the NDVI file according to the resolution of the DTM provided. All the layers, exception made for Landslide Inventory, had to be rasterized with the same resolution as the DTM.

Secondly, we had to extract some key factors from the DTM, more precisely the aspect, slope, plan and profile curvatures.

Afterwards, we had to use the raster calculator in order to find the No Landslide Zones, meaning those areas where the slope was lower than 20° or higher than 70°. With the resulting areas, we decided to remove some of the smaller areas. In our case we tried with 10, 50, 70 and 100 pixels and settled with 50 in the end, using the Sieve tool on QGIS. The resulted raster was vectorized in order to obtain polygons.

Once we had vectorized the raster, we removed the No Landslide Zones that were overlapping the Landslide Inventory polygons.

In the second part of the preprocessing of data we had to settle on a ratio of data to use as training-testing points for a machine learning model based on our dataset. The ratio selected was 70/30 (training/testing). To do this selection we used the tool Random Selection. In the end, we merged the processed landslide inventory layer with the No Landslide Zone one.

To create random points inside the polygons, we had to create 350 0-hazard and 350 1-hazard points (150 for both the testing dataset). This was obtained by using the Select Features by Value tool and making a selection based on the Hazard and Train_Test fields. Eventually, we merged separately the two kind of point layers (testing/training).

We then sampled the environmental factors with the training and testing point layers using the Point Sampling Tool plugin that we had installed beforehand.

After everything was done, we were left with two layers: trainingPointsSampled and testingPointsSampled

Step 2: Susceptibility Map Generation



In the second step we generated the susceptibility map starting from the results obtained in the previous step.

To do this we used the ModelMap R tool for QGIS and as a machine learning model we chose Random Forest, that through the use of training points and rasters of some environmental factors (Aspect, DTM, Distance to rivers, Distance to roads, Distance to faults, NDVI, Profile Curvature, Plan Curvature and DUSAF) generated the susceptibility map of our area.

Subsequently, we reclassified the newly obtained layer into two classes, 0 = [0 - 0.5) and 1 = [0.5 - 1), and we continued with the validation through the accuracy assessment and sampling Python script, using the reclassified susceptibility map and the testing points. We used said reclassified layer only in this step, while in the following ones the original susceptibility map will be used.



As a result of the validation we have obtained the following error matrix, where we can observe a precision index value of 0.73, a high enough value that makes the results of the susceptibility map more reliable.

Step 3: Data Preprocessing for Exposure Assessment



Before carrying out the exposure assessment, we had to process the data in order to make them compatible for the next operation.

As a first operation we reprojected the population raster dataset using the same CRS of the other layers and we clipped it using the vector layer of our area.

After that we reclassified the susceptibility map layer into 4 classes according to the landslide risk: [0, 0.25) = low; [0.25, 0.5) = moderate; [0.5, 0.75) = high; [0.75, 1] = very high. In the end, we resampled the susceptibility map layer using the resolution of the clipped population raster dataset: ~ 78 m.

With the layers just obtained we could proceed to the exposure assessment.

Once we got to this point, we had to conclude our analysis in order to understand how many people living in our area inhabited a high risk area.
